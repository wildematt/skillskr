## Design Context

### Users
SkillSkr serves AI tooling power users who manage local skill libraries across Codex, Claude, Cursor, Windsurf, Agents, and project-level folders. They use the app as a desktop control surface to browse, inspect, edit, organize, and maintain skill files with minimal friction.

### Brand Personality
Calm, precise, native.

The interface should feel dependable and fast, closer to a polished desktop utility than a flashy dashboard. The emotional goal is focused confidence: users should feel like the system is orderly, legible, and under control even when handling many skills from many sources.

### Aesthetic Direction
Use a macOS-adjacent desktop utility language with HeroUI semantic tokens as the foundation. Preserve the current airy, rounded, file-manager-like layout in light mode, and extend it into a dark mode that feels intentional rather than inverted.

References inferred from the product:
- Native desktop productivity tools with clear spatial separation
- Calm editorial density over neon "AI" styling

Anti-references:
- Cyberpunk dark UI with glow-heavy accents
- Glassmorphism-heavy cards and blurred gimmicks
- Generic SaaS dashboards with loud gradients everywhere

### Design Principles
1. Follow HeroUI theme semantics first: background, surface, overlay, default, accent, separator, and state colors should drive the visual system.
2. Keep elevation legible and restrained: browsing panes, detail panels, menus, and modals should separate by tone and border before relying on heavy shadow.
3. Reserve color for meaning: accent is for focus and selection, status colors are for state, and most of the interface should remain neutral and calm.
4. Optimize for scanning speed: dense content should remain easy to parse with strong contrast, clear hover/selected states, and minimal decorative noise.
5. Dark mode must feel designed, not inverted: use tuned dark neutrals, softer borders, and stable contrast while preserving the current desktop utility character.
